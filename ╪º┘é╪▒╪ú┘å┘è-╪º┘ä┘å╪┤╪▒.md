# حزمة Netlify — تطبيق بنك المعلومات المؤسسي

هذه حزمة **التطبيق فقط**. قاعدة البيانات في حزمة `ckb-supabase` المنفصلة —
جهّزها أولاً ثم انشر هذا التطبيق.

## خطوات النشر

1. ارفع هذا المجلد إلى مستودع GitHub جديد:

   ```bash
   git init && git add . && git commit -m "CKB v1.0"
   git branch -M main
   git remote add origin https://github.com/<حسابك>/ckb.git
   git push -u origin main
   ```

2. في Netlify: **Add new site → Import an existing project → GitHub**
   واختر المستودع. ملف `netlify.toml` موجود جاهزاً — لا تغيّر أوامر البناء.

   > ⚠️ لا تستخدم السحب والإفلات (Netlify Drop) — التطبيق فيه كود خادم
   > ويجب أن يبنيه Netlify عبر Git.

3. قبل Deploy أضف في **Site configuration → Environment variables**
   (القيم من حزمة Supabase → Project Settings → API):

   | المتغير | القيمة |
   |---|---|
   | `NEXT_PUBLIC_SUPABASE_URL` | Project URL |
   | `NEXT_PUBLIC_SUPABASE_ANON_KEY` | مفتاح anon |
   | `SUPABASE_SERVICE_ROLE_KEY` | مفتاح service_role (سري) |

4. **Deploy site** ← افتح الرابط وسجّل الدخول بحساب مدير النظام
   الذي أنشأته في خطوات حزمة Supabase.

للتشغيل محلياً: `cp .env.example .env.local` وعبّئ نفس القيم الثلاث،
ثم `npm install && npm run dev`.
